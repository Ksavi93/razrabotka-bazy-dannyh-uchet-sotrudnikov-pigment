# Employee44 — система учета сотрудников предприятия

## Описание проекта

**Employee44** — настольное приложение для учета сотрудников предприятия, разработанное на языке Python с использованием библиотеки `tkinter` для графического интерфейса и `sqlite3` для хранения данных.

Приложение предназначено для:
- хранения сведений о сотрудниках;
- добавления новых записей;
- просмотра данных в табличном виде;
- редактирования карточки сотрудника;
- удаления сотрудника с переносом записи в архив;
- поиска и сортировки по столбцам;
- быстрой проверки актуальности базы данных.

Проект выполнен в рамках практической части дипломной работы на тему разработки базы данных для учета сотрудников предприятия.

## Функциональность

- Авторизация пользователя.
- Просмотр списка сотрудников в `Treeview`.
- Добавление новых сотрудников.
- Редактирование карточки сотрудника.
- Удаление сотрудника с архивированием причины удаления.
- Поиск по всем полям.
- Сортировка по колонкам таблицы.
- Обновление таблицы после операций.

## Требования к запуску

Для работы проекта требуется:
- установленный Python 3.10 или выше;
- стандартная библиотека `tkinter` (обычно входит в состав Python);
- доступ на запись в папку проекта для создания файла базы данных SQLite.

Дополнительные библиотеки не требуются.

## Установка и запуск

1. Склонировать репозиторий:
   ```bash
   git clone <repository-url>
   cd employee44
   ```

2. При необходимости создать виртуальное окружение:
   ```bash
   python -m venv venv
   ```

3. Активировать окружение:
   - Windows:
     ```bash
     venv\Scripts\activate
     ```
   - Linux / macOS:
     ```bash
     source venv/bin/activate
     ```

4. Запустить приложение:
   ```bash
   python main.py
   ```

При первом запуске автоматически создается база данных `data/employee44.db` и таблицы проекта.

## Проверка работоспособности

После запуска рекомендуется проверить следующие сценарии:
1. Успешная авторизация.
2. Добавление нового сотрудника.
3. Открытие карточки сотрудника двойным щелчком по строке таблицы.
4. Изменение и сохранение данных сотрудника.
5. Удаление сотрудника с указанием причины и даты.
6. Поиск сотрудников по части строки.
7. Сортировка данных по столбцам таблицы.
8. Обновление списка сотрудников после изменений.

## Сценарии тестирования

### 1. Проверка авторизации
- Ввести корректные логин и пароль.
- Убедиться, что открывается основное окно приложения.

### 2. Проверка добавления записи
- Заполнить все обязательные поля формы.
- Сохранить запись.
- Проверить, что сотрудник появился в таблице.

### 3. Проверка редактирования
- Открыть карточку сотрудника.
- Изменить одно или несколько полей.
- Сохранить изменения.
- Убедиться, что данные обновились.

### 4. Проверка удаления и архивации
- Ввести ID сотрудника, причину удаления и дату.
- Удалить запись.
- Проверить, что сотрудник исчез из основной таблицы.
- Проверить, что запись добавилась в таблицу `ARCHIVE`.

### 5. Проверка поиска
- Ввести фрагмент фамилии, должности или другого поля.
- Убедиться, что таблица отфильтровалась по совпадениям.

### 6. Проверка сортировки
- Нажать на заголовок столбца.
- Убедиться, что порядок строк изменился.

## Примечания

- Пароль и логин в проекте демонстрационные.
- При необходимости базу данных можно очистить, удалив файл `data/employee44.db`.
- Для хранения фотографий сотрудников используется папка `photo/`.
